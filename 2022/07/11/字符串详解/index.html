String

子串和子序列

子串：要求连续的一段

子序列：从原字符串中提取出来的相对位置没有改变的序列

STL

  • 重载了赋值运算符 +，当 + 两边是 string/char/char[]/const char* 类型时，可以将这两个变量连接，返回连接后的字符串（string）。
  • 赋值运算符 = 右侧可以是 const string/string/const char*/char*。
  • 访问运算符 [cur] 返回 cur 位置的引用。
  • 访问函数 data()/c_str() 返回一个 const char* 指针，内容与该 string 相同。
  • 容量函数 size() 返回字符串字符个数。
  • find(ch, start = 0) 查找并返回从 start 开始的字符 ch 的位置；rfind(ch) 从末尾开始，查找并返回第一个找到的字符 ch 的位置（皆从 0 开始）（如果查找不到，返回 -1）。
  • substr(start, len) 可以从字符串的 start（从 0 开始）截取一个长度为 len 的字符串（缺省 len 时代码截取到字符串末尾）。
  • append(s) 将 s 添加到字符串末尾。
  • append(s, pos, n) 将字符串 s 中，从 pos 开始的 n 个字符连接到当前字符串结尾。
  • replace(pos, n, s) 删除从 pos 开始的 n 个字符，然后在 pos 处插入串 s。
  • erase(pos, n) 删除从 pos 开始的 n 个字符。
  • insert(pos, s) 在 pos 位置插入字符串 s。
  • std::string 重载了比较逻辑运算符，复杂度是 $\Theta(N)$ 的。

匹配后缀和前缀

最短公共前后缀长度

而每次找最小公共前后缀，可以直接循环直next[j] == 0

然而这题数据范围较大，每次求最小公共前后缀的时间复杂度可能退化为O(n2)，所以可以记忆化，每次使得ne[i]=j

KMP本质

$对于一个字符串str，第i位，已知ne[i]数组表示str[0-i]的最长公共前后串的长度$

$不难发现，ne[i],ne[ne[i]]…ne[ne[i]^t]都是字符串str[0-i]的公共前后串，只不过是ne[i]是其中最大的罢了$

$因此我们其实可以得到str[0-i]的公共前后串个数为num[i] = num[ne[i]] + 1 = num[ne[ne[i]]] + 2$

后缀排序

这个是未优化版本的，先从这讲起

后缀排序一步步优化的思想其实是比较简单和明了的，但是代码很绕

最先想到的算法肯定是对所有后缀字符串进行qsort，这样的算法复杂度比较高$O(N^2log(N))$

可以想到优化这个过程，用倍增法优化

先对每个后缀的第一关键字进行排序（可以直接按照该位置字符的ascii）

排序结束后，得到排名，然后再考虑它的第二关键字，也就是按照rk[x]先排序，想同时按照rk[x + 1]排序

这时的排序都是考虑到第i个字符和i+1个字符一起的

然后是否继续考虑第三个关键字呢？不，直接考虑3和4，因为此时i+2的排序已经是考量了i+2和i+3了

依次类推，可以优化整个过程，使得复杂度降到$O(Nlog^2(N))$

这样还是太大了，我们可以用基数排序的方式优化排序过程，使得复杂度再度下降到$O(Nlog(N))$

给你一个字符串，每次从首或尾取一个字符组成字符串，问所有能够组成的字符串中字典序最小的一个。

height数组

LCP（最长公共前缀）

两个字符串S和T的LCP就是最大的x$(x\le min(S,T))使得S_i=T_i(1\le i \le x)$

该篇章中lcp(i, j)代表后缀i和后缀j的LCP长度

height数组定义

height[i]=lcp(sa[i], sa[i - 1])，也就是第i名的后缀和它前一名的后缀的最长公共长度

有引理：

$height[rk[i]]\ge height[rk[i - 1]]-1$

https://zhuanlan.zhihu.com/p/408261126

证明：

$若height[rk[i-1]]\le 1$显然

$若height[rk[i-1]]\ge 1$，则考虑i-1后缀为cXZ，则其字符串意义上的下一位是XZ，设其字典序意义上的上一位为cXY（这里有$height[rk[i-1]]\ge 1$的保证），而因为知道了至少有XY的存在，所以i的字典序意义上的下一位一定有前缀X，所以得证

应用
求两子串最长公共前缀

对于两个子串$S_1:a_l….a_r和S_2:b_l…b_r$，不妨设$rk[a_l]<rk[b_l]，令rk[a_l]=i,rk[b_l]=j$

则$LCP(S_1,S_2)=min(\left| S_1 \right|, \left| S_2 \right|, min_{k=i+1}^jheight(k))$

证明：

求一个字符串的两个子串的大小关系

并不是简单看得那种比较第一个字符的方法（有可能前很多个都一样）

假设需要比较的是：$A=S[a…b],B=S[c…d]$的大小关系

若$LCP(a,c)\ge min(\left| A \right|, \left| B \right|),则A < B \iff \left| A \right| < \left| B \right| $

否则$A<B\iff rk[a]<rk[c]$

统计不同子串的数目

把子串当成后缀的前缀，所以可以枚举每个后缀，计算前缀的个数，再减去重复值

前缀的总个数就是n(n+1)/2

而对于重复值，如果按照后缀数组的顺序枚举，则每次新增的子串就是除去与上一个后缀（i-1）的LCP剩下之外剩下的前缀

去除的部分就是height[i]

所以结果就是$\frac {n(n+1)} 2-\sum^n_{i=2}height[i]$

出现至少k次的子串的最大长度

从后缀数组排序意义上，如果从第i个到第i+k个的后缀数组中，height的最小值=m大于等于0，说明这些后缀的前缀0…m相同，且重复了至少k次（不用考虑多余的了）

因此只要每次找区间大小为k的区间，然后对其中height的最小值取最大，这个过程可以用单调队列来做，即可获得答案

单调队列

这里的单调队列用multiset来做，比较方便

因为multiset内部是红黑树，它可以保证对于相同的值，是按照进入的顺序来存放的

因此s.erase(*find(i))可以保证删除第一个插入的权值为i的点

然后对于最大值或最小值，可以直接用$s.begin()$以及$s.end()$来操作

与单调栈相结合

https://loj.ac/p/2377

重点在于求$\sum_{1\le i<j\le n}LCP(T_i,T_j)$

问题可以转化为对于一个height[i]，求其左边有多少个点大于自己，右边有多少个点大于等于自己

这个问题可以用单调栈解决（预处理出来每个点的左端点和右端点）

先从左往右扫，栈里存放的点i越靠近栈顶hei[i]越大，当遇到一个新点j，则弹出所有hei[i]大于hei[j]的点，则剩下的点+1就是左端点

而右端点的求法同理

对答案的贡献就是$height[i](re[i] - i + 1)(i-le[i]+1)$

Manacher算法

用$d_1[i]$代表以i为中心，可以向两边延伸$d_1[i]$的回文串

用$d_2[i]$代表以i为回文串左中心，右端点为$i+d_2[i]-1$，左端点为$i-k$的回文串

用l，r来代表当前找到的回文串中，右端点最靠右的回文串

那么对于一个新点，若i>r，则按照朴素方法向两边扩展回文串，并记录l，r

若i<=r，则考虑因为l-r是回文串，则$d[i]\ge d[l+r-i]$，大于的情况出现于d[l+r-i]+i==r时，考虑在lr的范围之外可以继续扩展回文串，所以从r开始继续进行朴素扩展
